## Choosing the Right Connector

| Feature | Trino REST API | DBT | Analytics API |
|---------|---------------|-----|---------------|
| **Use Case** | Direct SQL queries in your tools | Data transformation pipelines | Programmatic access |
| **Query Language** | Standard SQL | DBT + SQL | SQL via HTTP |
| **Integration** | BI tools, SQL clients | DBT workflows | Applications, scripts |
| **Latency** | Low (direct connection) | Batch processing | Medium (async) |
| **Cost** | Credit-based per query | Credit-based per query | Credit-based per query |

## Trino Connector Quick Start

The Trino connector lets you query Dune data using any Trino-compatible client.

### 1. Get Your API Key

Get your API key from [dune.com/apis](https://dune.com/apis?tab=keys)

### 2. Configure Your Client

<CodeGroup>
```python Python (Trino Client)
from trino.dbapi import connect

conn = connect(
    host='sql.dune.com',
    port=443,
    http_scheme='https',
    auth=trino.auth.BasicAuthentication('x', 'YOUR_API_KEY'),
    catalog='dune',
    schema='ethereum'
)

cursor = conn.cursor()
cursor.execute('SELECT * FROM transactions LIMIT 10')
rows = cursor.fetchall()
```

```bash DBeaver / SQL Client
# Connection settings:
Host: sql.dune.com
Port: 443
Database: dune
Schema: ethereum
Username: x
Password: YOUR_API_KEY
```

```python Tableau
# In Tableau, select "Presto" connector
# (Trino is compatible with Presto protocol)

Server: sql.dune.com
Port: 443
Username: x
Password: YOUR_API_KEY
Catalog: dune
Schema: ethereum
```
</CodeGroup>

### 3. Run Queries

Query Dune data just like any other SQL database:

```sql
-- Get recent DEX trades
SELECT 
    blockchain,
    project,
    trader,
    amount_usd,
    block_time
FROM dex.trades
WHERE block_time > NOW() - INTERVAL '24' HOUR
LIMIT 100;

-- Analyze NFT sales
SELECT 
    collection,
    COUNT(*) as sales,
    SUM(amount_usd) as volume
FROM nft.trades
WHERE block_time > NOW() - INTERVAL '7' DAY
GROUP BY collection
ORDER BY volume DESC;
```

<Tip>
All of Dune's curated tables are available through the Trino connector. Browse available tables in the [Data Catalog](/data-catalog/overview).
</Tip>

## DBT Connector Quick Start

Use DBT to build transformation pipelines on Dune data.

### 1. Install the DBT Dune Adapter

```bash
pip install dbt-dune
```

### 2. Configure Your Profile

Create or update `~/.dbt/profiles.yml`:

```yaml
dune:
  target: prod
  outputs:
    prod:
      type: dune
      api_key: YOUR_API_KEY
      schema: dune
```

### 3. Create a DBT Project

```bash
# Initialize new DBT project
dbt init my_dune_project

# Navigate to project
cd my_dune_project
```

### 4. Build Models

Create models that transform Dune data:

```sql
-- models/daily_dex_metrics.sql
{{
  config(
    materialized='table'
  )
}}

SELECT 
    DATE_TRUNC('day', block_time) as date,
    blockchain,
    project,
    SUM(amount_usd) as daily_volume,
    COUNT(*) as daily_trades,
    COUNT(DISTINCT trader) as unique_traders
FROM {{ source('dex', 'trades') }}
WHERE block_time > NOW() - INTERVAL '90' DAY
GROUP BY 1, 2, 3
```

### 5. Run Your Pipeline

```bash
# Run all models
dbt run

# Test your models
dbt test

# Generate documentation
dbt docs generate
dbt docs serve
```

## Example: Building a BI Dashboard

Here's a complete example of connecting Tableau to Dune:

### Step 1: Connect to Dune

```
Connector: Presto
Server: sql.dune.com
Port: 443
Catalog: dune
Schema: ethereum
Username: x
Password: [YOUR_API_KEY]
```

### Step 2: Create Custom SQL

```sql
-- Custom SQL in Tableau
SELECT 
    DATE_TRUNC('day', block_time) as date,
    project as protocol,
    blockchain,
    SUM(amount_usd) as volume,
    COUNT(*) as trades,
    COUNT(DISTINCT trader) as users
FROM dex.trades
WHERE block_time > NOW() - INTERVAL '30' DAY
GROUP BY 1, 2, 3
```

### Step 3: Build Visualizations

- **Line chart:** Volume over time by protocol
- **Bar chart:** Top protocols by volume
- **Heatmap:** Activity by day of week
- **KPI cards:** Total volume, trades, unique users

### Step 4: Publish

Publish your dashboard to Tableau Server or Tableau Online for your team to access.

## Performance Tips

<AccordionGroup>
  <Accordion title="Use appropriate schemas">
    Connect to the specific blockchain schema you need (ethereum, polygon, etc.) to improve query performance
  </Accordion>

  <Accordion title="Leverage curated tables">
    Use Dune's pre-processed curated tables (like `dex.trades`, `nft.trades`) instead of raw tables when possible
  </Accordion>

  <Accordion title="Apply filters early">
    Always filter by `block_time` to limit the amount of data scanned:
    ```sql
    WHERE block_time > NOW() - INTERVAL '7' DAY
    ```
  </Accordion>

  <Accordion title="Use materialized views">
    For frequently-run queries, create materialized views to cache results
  </Accordion>

  <Accordion title="Optimize aggregations">
    Pre-aggregate data in your queries or DBT models before visualization
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Connection refused or timeout">
    - Verify your API key is correct
    - Check that you're using port 443 (not 8080)
    - Ensure your firewall allows HTTPS connections
    - Try using `x` as username (not your email)
  </Accordion>

  <Accordion title="Authentication failed">
    - Double-check your API key
    - Make sure there are no extra spaces or characters
    - Try regenerating your API key
  </Accordion>

  <Accordion title="Table not found">
    - Verify the schema name (ethereum, polygon, etc.)
    - Check table exists in [Data Catalog](/data-catalog/overview)
    - Use fully qualified table names: `dune.ethereum.transactions`
  </Accordion>

  <Accordion title="Query timeout">
    - Add time filters to reduce data scanned
    - Use curated tables instead of raw data
    - Consider using `LIMIT` for exploratory queries
    - Upgrade to large performance tier for complex queries
  </Accordion>
</AccordionGroup>
